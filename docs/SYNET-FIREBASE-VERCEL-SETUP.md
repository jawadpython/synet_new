# SYNET — Firebase & Vercel Setup Guide

Step-by-step guide to connect Firestore to the website and deploy on Vercel.

**Companion docs:** [SYNET-FIRESTORE-DATABASE.md](./SYNET-FIRESTORE-DATABASE.md)

---

## Overview

| Layer | Technology |
|-------|------------|
| Hosting | Vercel (Next.js 16) |
| Database | Firebase Firestore |
| Form writes | Next.js API routes + Firebase Admin SDK |
| Admin (future) | Firebase Auth + client SDK |

Public forms **never** write to Firestore from the browser. They POST to `/api/contact`, `/api/quote`, `/api/enrollment`, which use the Admin SDK server-side.

## Part 1 — Create Firebase project

> **Already done** for SYNET — project `synet-5603c`.

1. Go to [Firebase Console](https://console.firebase.google.com/) → **Add project**
2. Name: `synet-website` (or your choice)
3. Disable Google Analytics for now (optional — enable later for GA4)
4. Choose Firestore region: **`eur3` (Europe)** — closest to Morocco

### Enable Firestore

1. **Build → Firestore Database → Create database**
2. Mode: **Production** (rules file is in this repo)
3. Location: `eur3`

### Enable Authentication (for future admin)

1. **Build → Authentication → Get started** (one-time — initializes Auth config)
2. Enable **Email/Password** sign-in

> CLI cannot auto-enable Auth until you click **Get started** once in the Firebase Console.

---

## Part 2 — Service account (server writes)

1. **Project settings** (gear) → **Service accounts**
2. Click **Generate new private key** → download JSON
3. **Never commit this file.** Add to `.gitignore` if stored locally.

### Local development

Copy `.env.example` to `.env.local`:

```bash
cp .env.example .env.local
```

**Option A — single JSON variable (easiest):**

```env
FIREBASE_SERVICE_ACCOUNT_JSON={"type":"service_account","project_id":"your-id",...}
```

Paste the entire JSON on one line (Vercel uses the same format).

**Option B — separate variables:**

```env
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project-id.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
```

## Part 3 — Deploy Firestore rules & indexes

> **Already deployed** to `synet-5603c`. Re-run after rule changes:

```bash
npm run firebase:deploy
```

### Full CLI setup (new machine)

```bash
firebase login
npm run firebase:setup    # seeds settings + generates .env.local + service account key
npm run firebase:deploy   # rules + indexes
```

This deploys `firestore.rules` and `firestore.indexes.json`. Index builds can take a few minutes in the Firebase Console.

## Part 4 — Web app config (optional now, needed for admin later)

1. Firebase Console → **Project settings → Your apps → Add app → Web**
2. Register app name: `synet-admin`
3. Copy the `firebaseConfig` values into `.env.local`:

```env
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-id.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-id.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NEXT_PUBLIC_FIREBASE_APP_ID=...
```

These are **public** — safe in the browser. Admin SDK secrets stay server-only.

## Part 6 — Deploy to Vercel

### Connect repository

1. Push code to GitHub / GitLab / Bitbucket
2. [vercel.com/new](https://vercel.com/new) → Import repository
3. Framework: **Next.js** (auto-detected)
4. Root directory: `.` (project root)
5. Build command: `npm run build` (default)
6. Output: default

### Environment variables

In Vercel → **Project → Settings → Environment Variables**, add for **Production** and **Preview**:

| Variable | Value |
|----------|-------|
| `FIREBASE_SERVICE_ACCOUNT_JSON` | Full service account JSON (one line) |
| `NEXT_PUBLIC_SITE_URL` | `https://your-domain.vercel.app` or custom domain |

Optional (admin later): all `NEXT_PUBLIC_FIREBASE_*` variables.

### Deploy

Click **Deploy**. After build succeeds:

- Visit `https://your-app.vercel.app/api/health`
- Submit a test contact form
- Confirm documents appear in Firebase Console → Firestore

### Custom domain

1. Vercel → **Domains** → Add `synet.ma` (or your domain)
2. Update DNS at your registrar per Vercel instructions
3. Set `NEXT_PUBLIC_SITE_URL=https://www.synet.ma`

## Part 7 — Create admin user

Prerequisites: **Authentication → Get started** + **Email/Password** enabled in Firebase Console.

```bash
npm run firebase:create-admin
```

Or non-interactive:

```bash
ADMIN_EMAIL=admin@synet.ma ADMIN_PASSWORD='YourSecurePass1!' ADMIN_NAME='SYNET Admin' npm run firebase:create-admin
```

This script:
1. Creates the Firebase Auth user
2. Sets custom claims `{ role: "admin", active: true }`
3. Creates `users/{uid}` in Firestore

> The admin dashboard UI (`/admin`) is not built yet — the account is ready for when it is.

## What gets written on form submit

| Form | Firestore collections | Reference prefix |
|------|----------------------|------------------|
| Contact | `leads` | `LED-2026-0001` |
| Quote | `companies`, `leads`, `serviceRequests` | `DEV-2026-0001` |
| Enrollment | `registrations`, `leads` | `INS-2026-0001` |

Reference numbers use sharded counters in `counters/{year}_{type}`.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `firebase: not_configured` on health | Add env vars to Vercel; redeploy |
| `PERMISSION_DENIED` on form submit | Rules block client writes — forms must use API routes (already implemented) |
| `FAILED_PRECONDITION` on query | Deploy indexes: `npm run firebase:deploy` |
| Private key error on Vercel | Use `FIREBASE_SERVICE_ACCOUNT_JSON` or ensure `\n` in `FIREBASE_PRIVATE_KEY` |
| Build succeeds but forms don't persist | Check Vercel **Functions** logs for `[contact]` / `[quote]` errors |
